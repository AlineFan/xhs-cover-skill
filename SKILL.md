# 小红书封面生成 Skill

## 第一步：收集素材

如果用户已经提供了文字内容或图片，**直接进入第二步**，不要重复询问。

若用户没有提供任何信息，询问：

```
你好！我来帮你生成小红书封面 🎬

需要你提供：
1. **内容描述**（必需）：视频/笔记讲什么？粘贴字幕文本、或1-3句话描述
2. **封面素材图**（可选）：视频截帧或封面照片，上传到对话（不传也能做）
3. **偏好风格**（可选）：干货图文 / 图文卡片 / 人物出镜 / 全图出镜
```

---

## 第二步：分析内容

### 图片视觉分析（有图时必做）

观察封面图，提取：
- **画面主体**：人物/风景/产品/截图/日常场景？
- **情绪氛围**：温暖/活泼/治愈/励志/干货？
- **色调**：明亮/暗调/暖色/冷色？

### 文字内容分析（优先级从高到低）

1. **有字幕文本** → 提取核心信息点、关键词、视频主题
2. **有视频描述** → 理解内容方向、受众、价值点
3. **仅有图片** → 根据视觉推断内容类型

### 推荐风格选择

- 截图/产品/干货内容 → 推荐 **A 干货图文型**
- 本人出镜照片（人物+背景分离） → 推荐 **C 人物出镜型**
- 整张照片/风景/氛围大图 → 推荐 **D 全图出镜型**
- 多图展示/教程步骤 → 推荐 **B 图文卡片型**
- 纯文字无图 → 推荐 **A 干货图文型**（无图状态）

## 第三步：生成爆款标题

为封面生成以下文案（**直接在 HTML 里预填充，不要让用户自己填**）：

- **badge 标签**（6-12字，放封面顶部）：如 `🔥 干货分享 · AI工具` / `✨ 必看` / `📌 收藏`
- **主标题**（5-14字，用于封面大字）
- **副标题**（6-12字，辅助说明）
- **Style C/D 关键词×3**（每个2-6字，以 `✅` 开头）

### 标题风格标准

小红书封面标题的核心是**短、口语、有钩子**。避免套模板，优先写出听起来像人说的话。

**好标题的特点：**
- 短到极致，5-10字最佳，宁可短不要长
- 口语化、不完整句也没关系
- 直接抛结果或悬念，不解释、不煽情
- 可以不押韵、不对称，自然就好

**真实爆款标题参考（对照这个风格生成）：**
```
看完这两个问题前，别养虾      ← 悬念，不解释
如何手搓一段有效简历          ← 直接说动作
在年前做一个很能赚的万粉账号  ← 结果具体
原来满地都是挣钱的招数啊      ← 发现感
是个人都能找到工作            ← 短，口气强
不要觉得赚钱很丢脸            ← 价值观共鸣
流量体质如何练成              ← 简练，问句
找工作野路子                  ← 极短，勾好奇
去除AI味的网站设计skill       ← 直接命名工具/方法
神级skill，让UI从丑到美       ← 工具介绍：转变型结构
```

**内容类型 → 标题结构对照：**

| 内容类型 | 推荐结构 | 示例 |
|----------|----------|------|
| 教程/方法 | 直接说动作或结果 | `如何手搓一段有效简历` |
| 工具/skill 介绍 | [工具标签] + 从X到Y | `神级skill，让UI从丑到美` |
| 问题/避坑 | 悬念 or 反常识 | `看完这两个问题前，别养虾` |
| 经验/干货 | 极短 + 勾好奇 | `找工作野路子` |
| 观点/价值观 | 直接说出来 | `不要觉得赚钱很丢脸` |

**⚠️ 介绍工具/skill 类内容常见错误：**
- ❌ 问题导向（`为什么AI写的界面总差点味道`）→ 太绕，不够直接
- ✅ 转变导向（`神级skill，让UI从丑到美`）→ 直接点出价值，有对比感

**主标题可用 `<span class="accent">关键词</span>` 标记1-2个最核心的词（显示为强调色），其余文字保持默认色。**

## 第四步：填充封面模板

### 读取模板

读取此 skill 目录下的 `assets/cover-scene1.html`（完整交互封面工具，含4种风格）。

### 填充文案

用字符串替换更新以下 contenteditable 元素的 innerHTML：

**Style A（干货图文）：**
- `.sA-brand` → badge 标签文字
- `.sA-title` → 主标题（可含 `<br>` 换行和 `<span class="accent">` 强调词）
- `.sA-sub` → 副标题

**Style B（图文卡片，共4个排版布局）：**
- 每个布局里的 `.sBL-tag` → badge 标签（4处相同）
- 每个布局里的 `.sBL-title` → 主标题（4处相同）
- 每个布局里的 `.sBL-sub` → 副标题（4处相同）

**Style C（人物出镜）：**
- `#styleC .sC-super` → badge 标签文字
- `#styleC .sC-main` → 主标题（可含 `<br>` 换行，字号更大，建议8-12字）
- `#styleC` 内三个 `.sC-kw` → 关键词标签（每个以 `✅` 开头）
- `#styleC .sC-sub` → 副标题

**Style D（全图出镜）：**
- `#styleD .sC-super` → badge 标签文字（与 Style C 共用 `.sC-overlay` CSS）
- `#styleD .sC-main` → 主标题（可含 `<br>` 换行，建议8-12字）
- `#styleD` 内三个 `.sC-kw` → 关键词标签（每个以 `✅` 开头）
- `#styleD .sC-sub` → 副标题

> Style D 与 Style C 文案结构完全相同，区别在于 D 是全屏背景图（无独立人物区域）。填充时 C 和 D 分别用父选择器 `#styleC` / `#styleD` 区分，避免互相污染。

### 保存文件

将填充完毕的完整 HTML 保存为工作目录下的 `小红书封面.html`。

## 注意事项

- 主标题字数控制在 **8-16 字**（Style C 建议 8-12 字）
- `<span class="accent">词</span>` 标记的是强调色词，用于主标题中最关键的1-2个词
- 字幕文本如果很长（>500字），优先提取前1/3的内容（视频开头往往说明主题）
- 若用户没有提供内容，仅凭图片分析，需说明标题是基于画面推断的
- badge 标签根据内容类型调整（如 "😋 好吃" / "✈️ 旅行" / "💡 干货"）
